# Stratégie de victoire — Zindi Multilingual Health QA

Cible : > 0.768 weighted score (ROUGE-1 37% + ROUGE-L 37% + LLM-judge 26%)

## TL;DR — La décision data-driven

L'analyse complète des 29 815 exemples train + 6 686 val révèle 6 leviers actionnables, classés par impact estimé :

| Levier | Impact estimé | Implémentation |
|--------|---------------|----------------|
| Calibration longueur par langue | +0.02-0.04 | `src/length_config.py` (fait) |
| Fine-tuning multilingue Aya 32B / Llama 70B | +0.05-0.10 | `bash train.sh` après baseline |
| Templates par langue (surtout Eng_Eth) | +0.02-0.05 | `src/templates.py` (fait) |
| RAG topic-context | +0.01-0.04 selon test A/B | `src/rag.py` (fait) |
| Beam search + MBR selection | +0.02-0.03 | À implémenter post-baseline |
| Logit bias termes médicaux | +0.01-0.02 | À tester sur exp F |

## Décisions architecturales validées par l'analyse

### 1. Modèle unique vs un par langue ?

**Décision : UN seul modèle fine-tuné** sur les 29k exemples.

**Pourquoi** :
- 56% du test est en anglais (Eng_Uga 28%, Eng_Gha 19%, Eng_Ken 6%, Eng_Eth 2%)
- Les langues africaines sont 44% mais chacune est petite (Amh_Eth = 2.3% du test = 61 questions)
- Un modèle multilingue (Aya Expanse 32B ou Llama 3.3 70B) apprend les 8 subsets simultanément
- Coût d'entraînement de N modèles séparés > gain marginal

### 2. Aya Expanse 32B vs Llama 3.3 70B ?

**Décision : à mesurer en Phase 0 baseline** (expériences B/C vs D/E).

**Hypothèses à valider** :
- Aya 32B devrait gagner sur Lug_Uga, Swa_Ken, Amh_Eth (langues africaines natives)
- Llama 70B devrait gagner sur tout l'anglais (Eng_*)
- Si écart faible (<0.02), choisir Aya 32B (couvre mieux les langues low-resource)

### 3. RAG context — Aide ou nuit ?

**Décision : à mesurer en Phase 0** (B vs C, D vs E).

**Analyse** : la réponse train du même hash a ROUGE ≈ 0 avec la réponse val ground truth (sauf Aka_Gha 0.17). Donc le RAG ne pourra PAS être copié, il sert uniquement à :
1. Indiquer le bon registre / vocabulaire médical pour la langue
2. Donner un exemple de structure (template, bullets, longueur)

Si le RAG aide réellement = +0.02. S'il dégrade = ne pas l'utiliser.

## Insights critiques par langue (à exploiter)

### Eng_Eth (60 questions test, 2.3%)
- **57% des réponses train suivent le template** `"This is a question about, [STI]. [brief explanation]"`
- **Détection topic fonctionne** sur ~65% des questions test
- **Stratégie** : pour ces questions, fine-tuning + RAG devraient apprendre le template ; à défaut on peut hardcoder le template en fallback
- **Longueur cible : 24 mots médiane, max 36 tokens**

### Eng_Uga (744 questions, 28.4% — le plus gros segment)
- Templates STI fixes : `"Chlamydia also known as..."`, `"Genital Herpes or Herpes Simplex..."`
- Top-10 openings couvrent 15.4% des réponses
- Top trigrammes : `also known as`, `Herpes Simplex Virus (HSV)`, `Chlamydial Genitourinary`
- **Longueur cible : 73 mots médiane, max 112 tokens**

### Aka_Gha (492 questions, 18.8%)
- Réponses longues et en prose (médiane 100 mots)
- Vocabulaire concentré : `nna ne awo` (sexuel et reproductif), `awo akwahosan ho` (santé reproductive)
- Top-10 openings couvrent seulement 2.6% → beaucoup de diversité, fine-tuning indispensable
- **Longueur cible : 100 mots médiane, max 182 tokens**

### Lug_Uga (374 questions, 14.3%)
- `Akawuka akaleeta Siriimu` (HIV) = 2 362 occurrences dans train ⚡
- Templates STI : `Kiramidiya era amanyiddwa nga Akawuka`, `Eddagala eritta bbakitiiriya erya Penicillin`
- **Longueur cible : 68 mots médiane, max 123 tokens**

### Swa_Ken (229 questions, 8.7%)
- `virusi vya ukimwi` (HIV) = 1 755 occurrences ⚡
- Templates STI : `virusi vya ukimwi`, `Upungufu wa Kinga`
- **Longueur cible : 66 mots médiane, max 120 tokens**

### Amh_Eth (61 questions, 2.3%)
- Réponses **très courtes** (médiane 19 mots — moitié de l'anglais Ethiopia)
- Vocabulaire concentré : `ኤች አይ ቪ` (HIV) = 54 occurrences
- ⚠ 243 réponses train contiennent du Latin (probablement termes médicaux comme HIV/PrEP)
- **Longueur cible : 19 mots médiane, max 34 tokens** — critique car le moindre dépassement tue ROUGE

### Eng_Gha (491 questions, 18.8%)
- Top trigramme : `sexual and reproductive` (1 118 occurrences)
- Patterns : `Here are some...`, `to make informed decisions`, `the importance of`
- **Longueur cible : 70 mots médiane, max 107 tokens**

### Eng_Ken (167 questions, 6.4%)
- Top trigramme : `the risk of` (501), `important to note that` (216)
- `HIV (Human Immunodeficiency Virus)` revient 302 fois (template)
- **Longueur cible : 64 mots médiane, max 98 tokens**

## Anomalies détectées (à corriger en post-processing)

- **1 469 questions train dupliquées** : peuvent contenir des réponses différentes pour la même question → augmentation gratuite
- **243 réponses Amharic avec caractères Latin** : généralement termes médicaux légitimes (HIV, PrEP), ne pas filtrer aveuglément
- **17 réponses Aka_Gha potentiellement code-switchées** : surveiller en output
- **5 réponses train de moins de 2 mots** : probablement bruit, à filtrer du training

## Plan d'exécution Mac Studio (révisé après analyse)

### Phase 0 — Baseline (Mac Studio, ~2-3h)
```bash
bash setup.sh        # ~30-50 min
bash baseline.sh     # 6 expériences sur 200 val examples
```
**Output attendu** : `outputs/baselines/summary.json` avec ranking des 6 setups.

**Critères de décision après Phase 0** :
- Si Aya 32B sans RAG ≈ Llama 70B sans RAG → **Aya 32B** (plus petit, plus rapide à fine-tuner)
- Si écart > 0.03 → modèle gagnant
- Si avec_RAG > sans_RAG → utiliser RAG en fine-tuning et inférence
- Si length_cal > sans_cal → toujours activer

### Phase 1 — Fine-tuning (Mac Studio, ~1-3h)
```bash
bash train.sh
```
Avec :
- Modèle décidé en Phase 0
- LoRA rank 16, 3000 itérations
- Seed 42, save every 500 steps
- Validation sur 500 val examples toutes les 200 steps

### Phase 2 — Soumission (Mac Studio, ~30 min)
```bash
bash infer.sh
```
Avec :
- Length calibration stricte par langue (conservative=True)
- RAG context si Phase 0 le valide
- Beam search 4 (compromise vitesse/qualité)

### Phase 3 — Itération
Si score < 0.768 :
1. Analyser les erreurs par langue dans `outputs/analysis/`
2. Augmenter dataset si une langue underperforme
3. Tester logit bias sur termes médicaux clés (`templates.py`)
4. Tester ensemble Aya+Llama si Phase 0 montre des forces complémentaires

## Anti-patterns à éviter

1. ❌ **Ne pas fine-tuner avant baseline** — risque de gaspiller 2h sur un modèle sous-optimal
2. ❌ **Ne pas générer sans length calibration** — Amh_Eth en particulier tue le score si > 30 mots
3. ❌ **Ne pas oublier seed=42** — disqualification possible si non-reproductible
4. ❌ **Ne pas optimiser un seul modèle pour les 8 langues sans validation per-language** — risque de boost moyen + dégradation sur une langue
5. ❌ **Ne pas générer en greedy seul** — beam search >3 obligatoire pour ROUGE
